基本配置

webpack是用nodejs写的一个模块，使用webpack，其实是使用nodejs的模块，使用nodejs的包管理命令，在一个空文件夹中初始化一个项目

npm init -y

其中，“-y”表示使用默认设置，这样可以在文件夹中创建一个package.json的包配置文件，在项目文件夹中再创建一个src文件夹和一个public文件夹，其中，src文件夹放置开发文件，public文件夹放置打包后的文件，在src文件夹中创建一个index.js文件，作为webpack的入口文件，这是webpack的通用套路。

接着，可以安装webpack相关模块：

npm i webpack webpack-cli -D

其中，“i”表示install的简写，webpack-cli模块的作用是可以让我们在命令行执行webpack命令，“-D”是“--save-dev”的简写，将包名称存储到package.json中的“devDependencies”设置项中。

此时，我们可以在index.js中写一点伪代码，然后就可以使用webpack打包了，执行下面的命令：

npx webpack ./src/index.js

其中，“npx”表示执行当前项目中安装的模块包命令，整个命令表示使用webpack打包index.js，命令执行完之后，项目文件夹中会多出一个dist文件夹，文件中会生成一个main.js文件，它就是打包后生成的文件，所以我们知道，webpack默认会将打包的文件放入dist文件夹中，同时它的名称默认是main.js，如果我们想设置这些默认值，可以创建一个webpack.config.js文件，在文件中设置初始配置：

const path = require('path');

module.exports = {
    entry:'./src/index.js',
    output:{
        filename:'bundle.js',
        path:path.resolve(__dirname,'public')
    }
}

其中，entry表示入口文件，output表示打包后的文件名及存储地址，地址是通过path模块拼接生成的绝对地址，设置之后，删掉刚才的dist文件夹，再执行打包命令，此时，可以不带文件名，webpack会自动安装配置的文件执行：

npx webpack

命令结束后，打包的文件会自动放到public文件夹中，文件名为bundle.js。
我们还可以将这个命令配置到package.json的“scripts”之中

......
"scripts": {
    "dev": "npx webpack"
},
......

此时，可以这么执行打包命令： 

npm run dev

在打包过程中会提示没有设置“mode”选项，这个选项有两个值，一个是“development”，设置当前模式为开发模式，此时打包出来的js文件是没有压缩和混淆的，方便开发时排错；还有一个值是“production”，设置当前模式为生产模式，此时打包出来的js文件是压缩和混淆的，方便上线。我们可以在webpack.config.js中设置如下：

module.exports = {
    mode:'development',
    entry:'./src/index.js',
    output:{
        filename:'main.js',
        path:path.resolve(__dirname,'public')
    }
}
